## 1. Work Item Type: AI Story

### Purpose
Tracks a single decomposed story unit through the AI Software Factory automated pipeline. Created by the Story Decomposer stage. Moves through 8 states as the factory processes it.

### Identity
| Property | Value |
|---|---|
| Name | `AI Story` |
| Reference Name | `Custom.AIStory` |
| Color | `#0078D7` (Blue) |
| Icon | Story (icon: `icon-story`) |
| Description | A work item representing a feature unit processed through the AI Software Factory pipeline. |
| Parent Process | Inherited from Basic |

### State Machine

| # | State | Category | Description | Color |
|---|-------|----------|-------------|-------|
| 1 | Drafted | Proposed | Story created by the AI Story Decomposer | #E6E6E6 |
| 2 | In Planning | In Progress | Factory is generating the test plan | #FFCC00 |
| 3 | Plan Review | In Progress | Generated implementation plan under review | #FF8C00 |
| 4 | In Coding | In Progress | Code being generated by AI | #0078D7 |
| 5 | Code Review | In Progress | AI-generated code under review | #993399 |
| 6 | In Testing | In Progress | Tests being executed | #339933 |
| 7 | Approved | Resolved | All gates passed | #00B300 |
| 8 | Blocked | Removed | Revision limit hit; needs human intervention | #CC0000 |

### State Transitions

```
Drafted ──────────────────────────────────────┐
    │                                          │
    ▼                                          │
In Planning ──► Plan Review ──► In Coding ──► Code Review
    │              │               │              │
    │              │               │              │
    │              ▼               │              │
    │         (revision loop)      │              │
    │              │               │              │
    └──────────────┘               │              │
                                   ▼              ▼
                              In Testing ◄──── Code Review (approved)
                                   │
                                   ├──► Approved (all tests pass)
                                   │
                                   └──► Blocked (max revisions hit)
```

#### Allowed Transitions (detailed)

| From State | To State | Reason | Trigger |
|---|---|---|---|
| Drafted | In Planning | Begin processing | Factory pipeline starts |
| In Planning | Plan Review | Test plan generated | Factory completes test gen |
| Plan Review | In Coding | Plan approved | Review passes |
| Plan Review | In Planning | Plan needs revision | Review requests changes (revision loop) |
| In Coding | Code Review | Code generated | Factory completes code gen |
| Code Review | In Testing | Code approved | Code review passes |
| Code Review | In Coding | Changes requested | Code review requests changes (revision loop) |
| In Testing | Approved | All tests passed | All gates clear |
| In Testing | Blocked | Max revision count reached | Escalation |
| *Any state* | Blocked | Escalation | Human intervention required |
| Blocked | Drafted | Reset and retry | Human unblocks, resets |

## 2. Work Item Type: AI Verification

### Purpose
Records a single "Grill Me" comprehension verification session for one persona on one feature. Captures questions, scores, root cause analysis, and triggered document fixes.

### Identity
| Property | Value |
|---|---|
| Name | `AI Verification` |
| Reference Name | `Custom.AIVerification` |
| Color | `#339933` (Green) |
| Icon | Checkmark (icon: `icon-check`) |
| Description | Records a Grill Me comprehension verification session. One per persona per feature. |

### State Machine

| # | State | Category | Description |
|---|-------|----------|-------------|
| 1 | Pending | Proposed | Grill session not yet started |
| 2 | In Progress | In Progress | Grill session actively underway |
| 3 | Completed | Resolved | Grill session finished, scores recorded |
| 4 | Needs Revision | Removed | Sign-off failed; document(s) need updating |

### State Transitions

```
Pending ──► In Progress ──► Completed
                │               │
                │               ├──► (sign-off = Approved) → Done
                │               │
                │               └──► (sign-off = Needs Revision) → Needs Revision
                │                                        │
                └────────────────────────────────────────┘
                     (documents revised, re-grill)
```

#### Allowed Transitions

| From State | To State | Reason | Trigger |
|---|---|---|---|
| Pending | In Progress | Grill started | Persona opens grill session |
| In Progress | Completed | Grill finished | All questions answered |
| Completed | Needs Revision | Sign-off failed | SignOffStatus = Needs Revision |
| Needs Revision | In Progress | Re-grill | Documents revised, retry |

## 5. State Machines (Detailed)

### 5.1 AI Story State Behavior

```yaml
states:
  Drafted:
    category: Proposed
    color: "#E6E6E6"
    description: Story created by the AI Story Decomposer
    field_rules:
      - StoryContext: required
      - ModelUsed: required
      - AcceptanceCriteria: optional
      - RevisionCount: default(0)
      - TestPlan: hidden
      - ImplPlan: hidden
      - PlanReviewStatus: hidden
      - CodeReviewStatus: hidden
      - TestResults: hidden

  In Planning:
    category: In Progress
    color: "#FFCC00"
    description: Factory generating the test plan and implementation plan
    field_rules:
      - TestPlan: editable
      - StoryContext: read-only
      - RevisionCount: read-only

  Plan Review:
    category: In Progress
    color: "#FF8C00"
    description: Implementation plan under human or AI review
    field_rules:
      - ImplPlan: editable
      - PlanReviewStatus: editable (required if transitioning out)
      - RevisionCount: editable (incremented by factory)
      - TestPlan: read-only

  In Coding:
    category: In Progress
    color: "#0078D7"
    description: AI generating code for this story
    field_rules:
      - StoryContext: read-only
      - TestPlan: read-only
      - ImplPlan: read-only
      - CodeReviewStatus: hidden

  Code Review:
    category: In Progress
    color: "#993399"
    description: AI-generated code under review
    field_rules:
      - CodeReviewStatus: editable (required if transitioning out)
      - RevisionCount: editable (incremented by factory)
      - StoryContext: read-only

  In Testing:
    category: In Progress
    color: "#339933"
    description: Automated tests being executed
    field_rules:
      - TestResults: editable

  Approved:
    category: Resolved
    color: "#00B300"
    description: All quality gates passed
    field_rules:
      - All fields: read-only

  Blocked:
    category: Removed
    color: "#CC0000"
    description: Revision limit exceeded; human intervention required
    field_rules:
      - All fields: editable (for reset/remediation)
```

### 5.2 AI Verification State Behavior

```yaml
states:
  Pending:
    category: Proposed
    color: "#E6E6E6"
    description: Grill session not yet started
    field_rules:
      - Persona: required
      - Harness: required
      - FeatureName: required
      - ModelUsed: required
      - QuestionsAsked: default(0)
      - FullyCorrect: default(0)
      - MinorGaps: default(0)
      - MajorMisunderstandings: default(0)
      - Score fields: hidden

  In Progress:
    category: In Progress
    color: "#FFCC00"
    description: Grill session actively underway
    field_rules:
      - GrillTranscript: editable
      - QuestionsAsked: editable
      - Persona: read-only

  Completed:
    category: Resolved
    color: "#339933"
    description: Grill session finished, scores recorded
    field_rules:
      - FullyCorrect: required
      - MinorGaps: required
      - MajorMisunderstandings: required
      - SignOffStatus: required
      - GrillDate: required (auto-set)
      - MisconceptionDetails: required
      - TriggeredDocFixes: optional
      - Other fields: read-only

  Needs Revision:
    category: Removed
    color: "#CC0000"
    description: Sign-off failed, documents need updating
    field_rules:
      - MisconceptionDetails: editable
      - TriggeredDocFixes: editable
      - SignOffStatus: read-only
```

## 6. Relationship Model

```
Epic (System)
  │
  ├── AI Story (Custom.AIStory) ── "Child" of Epic via System.Parent
  │     │
  │     ├── [Related] AI Verification (DEV) via "Story-Verification" link type
  │     └── [Related] AI Verification (QA) via "Story-Verification" link type
  │
  ├── AI Verification - BA/PO (Custom.AIVerification) ── "Child" of Epic
  │     └── [Related] via "Feature-Verification" link type
  │
  ├── AI Verification - UI/UX (Custom.AIVerification) ── "Child" of Epic
  │     └── [Related] via "Feature-Verification" link type
  │
  ├── AI Verification - DEV (Custom.AIVerification) ── "Child" of Epic
  │     └── [Related] via "Feature-Verification" link type
  │
  ├── AI Verification - QA (Custom.AIVerification) ── "Child" of Epic
  │     └── [Related] via "Feature-Verification" link type
  │
  └── Task (System) ── "Child" of AI Story
        └── Implements broken-down factory tasks
```

### Link Types Needed

| Link Type | Topology | Usage |
|---|---|---|
| `System.Parent` (built-in) | Hierarchy | Epic → AI Story, Epic → AI Verification, AI Story → Task |
| `Custom.VerificationLink` | Related (directed) | AI Story → AI Verification (DEV, QA) |
| `Custom.VerificationLink` | Related (directed) | Epic → AI Verification (BA/PO, UI/UX) |

### Naming Convention

Each work item title follows this convention:
- **AI Story:** `[AI] {EpicShortName} - {StoryName}`
- **AI Verification:** `[Grill] {Persona} - {FeatureName} - {Date}`

## 8. Pipeline Field Mapping

### Factory Stage → Field Writes/Reads

| Factory Stage | Writes To | Reads From | Condition |
|---|---|---|---|
| **Story Decomposer** | `AI Story.StoryContext`, `AI Story.AcceptanceCriteria`, `AI Story.ModelUsed`, `AI Story.RevisionCount = 0`, sets State = Drafted | Epic (via Parent link) | Upon Epic approval |
| **Test Plan Generator** | `AI Story.TestPlan`, sets State = In Planning | `AI Story.StoryContext`, `AI Story.AcceptanceCriteria` | From Drafted |
| **Plan Generator** | `AI Story.ImplPlan`, sets State = Plan Review | `AI Story.StoryContext`, `AI Story.TestPlan` | From In Planning |
| **Plan Reviewer** | Reads `AI Story.ImplPlan`, writes `AI Story.PlanReviewStatus` | If Approved → sets State = In Coding; if Revisions Requested → increments `AI Story.RevisionCount`, sets State = In Planning | |
| **Code Generator** | Sets State = In Coding | `AI Story.StoryContext`, `AI Story.ImplPlan` | Writes code to repo |
| **Code Reviewer** | Writes `AI Story.CodeReviewStatus` | Implementation plan, generated code | If Approved → State = In Testing; if Changes Requested → incr `AI Story.RevisionCount`, State = In Coding |
| **Test Executor** | Writes `AI Story.TestResults`, sets State = In Testing | `AI Story.TestPlan`, code | |
| **Gate Checker** | If all pass → State = Approved; If `AI Story.RevisionCount > 2` → State = Blocked | `AI Story.TestResults`, `AI Story.RevisionCount` | |

### Grill Me Stage → Field Writes

| Grill Stage | Writes To | Notes |
|---|---|---|
| **Session Created** | `AI Verification.Persona`, `AI Verification.Harness`, `AI Verification.FeatureName`, `AI Verification.ModelUsed`, State = Pending | |
| **Session Started** | State = In Progress | |
| **Question Asked/Answered** | `AI Verification.GrillTranscript` (appended), `AI Verification.QuestionsAsked` (incremented) | |
| **Scoring** | `AI Verification.FullyCorrect`, `AI Verification.MinorGaps`, `AI Verification.MajorMisunderstandings` | |
| **Sign-off** | `AI Verification.SignOffStatus`, `AI Verification.GrillDate` | |
| **Analysis** | `AI Verification.MisconceptionDetails`, `AI Verification.TriggeredDocFixes` | |
| **If SignOffStatus = Approved** | State = Completed | Pipeline continues |
| **If SignOffStatus = Needs Revision** | State = Needs Revision | Documents updated, re-grill |

## 9. Implementation Steps

### Phase 1: Create Custom Fields (in order)

These must be created BEFORE the work item types that use them.

1. Create each **picklist (global list)** via the Process Contracts/Lists REST API
2. Create each **field** via the Process/Fields REST API
3. Create the **work item types**
4. Add **states** to the work item types
5. Add **field rules** (when each field is editable/required)
6. Configure **form layout**

### Order of Operations

| Step | Action | API Endpoint | Method |
|---|---|---|---|
| 1 | Create picklists (PlanReviewStatus, CodeReviewStatus, SignOffStatus, Persona, Harness, AIFoundryModel) | `POST /_apis/work/processes/{processId}/lists` | POST |
| 2 | Create plain string fields (PlanReviewStatus, CodeReviewStatus, ModelUsed, Persona, Harness, FeatureName, ParentFeature) | `POST /_apis/work/processes/{processId}/fields` | POST |
| 3 | Create HTML fields (StoryContext, TestPlan, ImplPlan, TestResults, AcceptanceCriteria, MisconceptionDetails, GrillTranscript, TriggeredDocFixes) | `POST /_apis/work/processes/{processId}/fields` | POST |
| 4 | Create integer fields (QuestionsAsked, FullyCorrect, MinorGaps, MajorMisunderstandings, RevisionCount) | `POST /_apis/work/processes/{processId}/fields` | POST |
| 5 | Create DateTime field (GrillDate) | `POST /_apis/work/processes/{processId}/fields` | POST |
| 6 | Create work item type "AI Story" | `POST /_apis/work/processes/{processId}/workItemTypes` | POST |
| 7 | Create work item type "AI Verification" | `POST /_apis/work/processes/{processId}/workItemTypes` | POST |
| 8 | Add states to AI Story | `POST /_apis/work/processes/{processId}/workItemTypes/{witRefName}/states` | POST |
| 9 | Add states to AI Verification | `POST /_apis/work/processes/{processId}/workItemTypes/{witRefName}/states` | POST |
| 10 | Add field-to-WIT mappings (field instances) | `POST /_apis/work/processes/{processId}/workItemTypes/{witRefName}/fields` | POST |
| 11 | Add state transition rules | `PATCH /_apis/work/processes/{processId}/workItemTypes/{witRefName}/states/{stateId}` | PATCH |
| 12 | Configure form layouts | `PATCH /_apis/work/processes/{processId}/workItemTypes/{witRefName}/layout` | PATCH |

## 10. REST API Payloads

### 10.1 Authentication

```bash
# Using Personal Access Token (PAT)
export AZURE_DEVOPS_ORG="https://dev.azure.com/agentmerlin"
export AZURE_DEVOPS_PROJECT="AI Software Factory"
export PROCESS_ID="7e34d370-3e5e-4dc6-add7-d96b9cea5f2e"
export PAT="<your-personal-access-token>"
export AUTH="Authorization: Basic $(echo -n \":$PAT\" | base64)"

# Helper function
invoke_api() {
  local method=$1
  local url=$2
  local data=$3
  curl -s -X "$method" \
    -H "$AUTH" \
    -H "Content-Type: application/json" \
    -d "$data" \
    "$AZURE_DEVOPS_ORG$url"
}
```
